Project Architecture

/ 系统架构双轨设计

Track A: Figma Plugin Figma 插件端

  • Powered by Qwen on SiliconFlow. 基于硅基流动平台调用 Qwen 大模型。
  • Main thread (code.js) bypasses iframe CORS limits. 主线程发起网络请求，完美绕过 iframe CORS 跨域限制。
  • Directly exports selected layers as Base64 PNG. 自动提取并转换选中图层为 Base64 PNG 格式。
  • Supports full bilingual UI (Chinese / English). 支持全局中英双语界面一键切换。

Track B: Web App Web 画布端

  • Vite + React + Konva + Express stack. 基于 Vite, React 与 Konva 的现代全栈架构。
  • Backend proxies xAI Grok for structured generation. Express 后端代理 xAI Grok，严格输出结构化 JSON 布局。
  • Renders editable components on a digital canvas. 在数字画布上动态渲染 UI 组件与导师建议面板。
  • Operates independently from Figma plugin keys. 环境独立，与 Figma 插件端的 API Key 配置互不干扰。

Figma Integration

/ 插件极速部署指引

Initialization Sequence 安装初始化

  1. Open Figma → Plugins → Development → Import plugin from manifest... 打开 Figma 菜单 → 插件 → 开发 → 从清单导入插件...
  2. Navigate to and select figma-ui-tutor/manifest.json 在本地目录中选择并导入 manifest 配置文件
  3. Run Plugin → access Settings → Input SiliconFlow API Key 运行插件 → 点击「设置」 → 填入你的 SiliconFlow 密钥
  4. Code modification: Plugins → Development → Reload 若修改了代码，需点击「重新加载」生效

System Architecture

/ 工程目录解析 
Intelligent-UI-Helper/
├── package.json              # Web scripts, React + Express deps (前端及服务依赖)
├── server/                   # Express API with Grok client (代理调用后端)
├── src/                      # React + Konva UI frontend (Web端视图层)
├── shared/                   # Zod schemas & prompts (前后端共用的校验及提示词)
├── figma-ui-tutor/
│   ├── manifest.json         # Figma plugin manifest (插件注册清单)
│   ├── code.js               # Main thread: fetch, export PNG (主线程逻辑)
│   └── ui-v3.html            # Plugin UI + SiliconFlow / Qwen logic (插件交互页面)
└── README.md                 # Documentation (系统技术说明文档)
